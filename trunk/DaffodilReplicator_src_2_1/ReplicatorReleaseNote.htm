Daffodil Replicator v2.1 Release Notes

 

This document is intended to provide you with a quick summary of the high-level features introduced in Replicator v2.1.

 

New features

 

·        Support for Firebird  to other Databases Replication:

In the current Replicator version, we have incorporated the support for Firebird database to other database replication. The version  provide the flexibility of Firebird data replication to heterogeneous database servers.

 

·        MySQL 5.1 Database Replication Support

      Heterogeneous Data Synchronization between MySQL and other databases     has been introduced.

 

·        Scope for different Schema

Replicate data between different schema of database.

 

Performance enhancements

 

·        XML file creation process is reviewed and optimized. The new process drastically reduces the time taken in capturing the insertion, deletion and updation of records from Replicator Shadow tables and transmitting it to target database server. Due to this all the replication operations could be performed in much lower time as compared to earlier Replicator versions.

 

 

Bugs which have been fixed

 

·        Bug has been fixed related to run the replicator over the internet.

·        Maximum open cursor exceeds.

·        Solution for Tags problem in XML files. With the earlier versions if columnName is same as row or tableName, parsing of XML file creates problem.

 

Updates in Documentation

 

The following documents have been revised to explain the major functionalities introduced in Daffodil Replicator v2.1:

 

  • Daffodil Replicator Developer’s Guide
  • Daffodil Replicator Console Guide

 

We recommend you to go through “Daffodil Replicator Developer’s Guide” for understanding various concepts and terminologies associated with Daffodil Replicator. For a detailed understanding of the working of Replicator GUI, please refer to “Daffodil Replicator Console Guide”. Both the guides contain the required information for beginners. It explains how to start and stop Replicator and the different ways to work with Replicator API and Console.
